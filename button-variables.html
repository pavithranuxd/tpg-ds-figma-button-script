Section B — Global vs Component-specific

Group 1 — Global (mapped/surface, text, icon, border)
SpeakerRule

"Group one: Global. These sit at the root of Mapped — surface, text, icon, border. If a component's colour spec matches one of these exactly, don't make a new token — alias straight to it."

SpeakerExample

"When a button border matches the shared border style, you alias the global border/primary/default token instead of making a new one. No duplication."

Example

border reuses root border/primary/default

Group 2 — Component-specific (mapped/component/*)
SpeakerRule

"Group two: Component-specific. When a component's colour doesn't match anything in the global set, you create it inside the component's own group. Button gets its own surface/brand, surface/secondary, surface/white states — hover, pressed, disabled — none of which exist globally."

Example

component/button/surface/brand, secondary, success, white

SpeakerDecision rule

"Spec matches global → alias global. Spec is custom to this component → create it inside component/<name>/..."
