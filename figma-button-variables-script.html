Section A — The chain (colour only)

This chain applies to colour only. Padding, gap, radius skip Primitive/Brand/Semantic and are created directly inside the component's Mapped group — see Section C.
Primitiveprimitives
"Start at Primitive. Raw values, no meaning attached — just a purple."Open primitives → colour/tpg-purple/500
Brandbrand
"Brand takes that raw purple and gives it a role in the TPG brand — primary-1, shade 500."brand → tpg/color/primary-1/500 aliasing primitive
Semanticsemantics
"Semantic gives it intent — this isn't 'purple' anymore, it's 'action, primary, default' in light mode."semantics → tpg/light/color/action/primary/default
Mappedmapped-tpg
"Last stop: Mapped — what components actually bind to. Colour lives in four groups at the root: surface, text, icon, border. These four are the global set, shared across every component."Expand mapped-tpg root groups

Section B — Global vs Component-specific

Group 1 — Global
Rulesurface / text / icon / border
"Group one: Global. These sit at the root of Mapped — surface, text, icon, border. If a component's colour spec matches one of these exactly, don't make a new token — alias straight to it."
Group 2 — Component-specific
Rulecomponent/button/*
"Group two: Component-specific. When a component's colour doesn't match anything in the global set, you create it inside the component's own group. Button gets its own surface/brand, surface/secondary, surface/white states — hover, pressed, disabled — none of which exist globally."Expand component/button/surface/*

Section B — Global vs Component-specific

Group 1 — Global (mapped/surface, text, icon, border)
Rule
"Group one: Global. These sit at the root of Mapped — surface, text, icon, border. If a component's colour spec matches one of these exactly, don't make a new token — alias straight to it."
Example
"When a button border matches the shared border style, you alias the global border/primary/default token instead of making a new one. No duplication."border reuses root border/primary/default
Group 2 — Component-specific (mapped/component/*)
Rule
"Group two: Component-specific. When a component's colour doesn't match anything in the global set, you create it inside the component's own group. Button gets its own surface/brand, surface/secondary, surface/white states — hover, pressed, disabled — none of which exist globally."component/button/surface/brand, secondary, success, white
Decision rule
"Spec matches global → alias global. Spec is custom to this component → create it inside component/<name>/..."
